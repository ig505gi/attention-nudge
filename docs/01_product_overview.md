# 产品概述与阶段计划

**Date**: 2026/04/11
**Status**: 规划阶段

---

## 产品定位

**AttentionNudge** — 一款基于 LLM 上下文理解的浏览器专注力辅助插件。

区别于传统的域名黑名单硬阻断或纯事后的时间统计，本产品通过"内容感知"实现动态的柔性干预。当用户陷入无意识的"信息流深渊"时，它化身心理教练，通过柔性提问打断用户的自动驾驶状态，将其拉回正轨。

核心差异化：
- **语义理解驱动**：LLM 实时比对"当前网页摘要"与用户的"专注目标"
- **柔性干预哲学**：非强制弹窗+交互式提醒，而非粗暴阻断
- **本地化复盘**：基于本地数据库生成每日注意力分布看板

---

## 目标用户

1. **核心用户**：开发者、创作者、研究者等重度脑力劳动者
2. **延伸用户**：有 ADHD 或专注困难倾向的学生/职场人
3. **付费转化目标**：不愿折腾 API Key 的"小白"用户

---

## 三阶段发展路线

### 阶段一：MVP 与种子用户验证（2-3 周）

**目标**：验证"柔性干预"核心痛点是否真实存在，打通主链路。

**功能范围**：
- 设置页：填入 API Key（支持 DeepSeek 等低成本模型）和 API URL
- 交互：点击插件图标，输入"当前任务"
- 后台：每次 URL 变化或定时，提取当前页面 Title + Meta，发给 LLM 比对
- 干预：不相关时弹出浏览器 Notification 或 DOM 注入弹窗
- 实时偏离指数（1-5 分）驱动不同强度干预

**不做**：复杂 UI、本地数据图表、键鼠活跃度监听

**发布渠道**：V2EX、掘金、少数派、B站

**形态**：免费、闭源（可录制 F12 演示视频自证隐私）、BYOK

---

### 阶段二：护城河构建（1-2 个月）

**目标**：根据用户反馈修复 Bug，上线真正能留住用户的"数据看板"。

**功能迭代**：
1. 引入 `chrome.idle` 增加"有效停留"判断，降低误报率
2. 核心护城河：接入 IndexedDB，存储每次检测记录
3. "今日复盘"页面：ECharts 画出注意力分布图
4. 用真实 Bad Case 数据优化 System Prompt

**用户反馈闭环设计**：
- 干预弹窗增加 👍/👎 反馈按钮
- 点击 👎 时询问"是否愿意匿名发送【网页标题+目标】用于改进"
- 通过 Webhook 收集至飞书/钉钉机器人或 Serverless 函数

---

### 阶段三：商业化与闭环

**目标**：为"小白"用户提供一键订阅服务。

**功能**：
- 引入国内支付渠道（面包多/微信收款码/个人支付接口）
- 提供无状态服务器网关，承担云端 API 费用
- 前端客户端保持开源

**海外扩展**（视国内留存率决定）：
- 英文本地化
- 接入 Stripe/LemonSqueezy/Paddle
- Product Hunt、Reddit、Twitter 发布

---

## 隐私与合规策略

1. **数据最小化原则**：只提取 `<title>`、`<meta description>`、首段文本
2. **BYOK 优先**：用户自备 API Key，数据直连大模型厂商
3. **本地存储**：IndexedDB 存储明细数据，不上传服务器
4. **无状态网关**：订阅模式下服务器仅做转发，不留存浏览历史
5. **开源客户端**：GitHub 公开插件前端代码，接受 Code Review
